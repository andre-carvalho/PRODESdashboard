Visão

O projeto TerraBrasilis refere-se a uma camada de serviços sobre um banco de dados geográfico, acessada através de um portal web, que vem sendo conduzido pelo INPE, visando melhorar o acesso e o uso dos dados produzidos nos programas de monitoramento ambiental pelos quais responde. Em um primeiro momento visa atender principalmente ao Programa Amazônia, mas o conceito e os pacotes de software produzidos podem ser adaptados e customizados para atender a outros projetos que produzam dados geográficos e que pretendem que esses possam ser acessados no ambiente da internet. Para a implementação do TerraBrasilis em projetos de monitoramento, deve-se considerar vários aspectos tecnológicos gerais, bem como as especificidades desse tipo de projeto. Dentre esses desafios podem ser citados os seguintes:

  • Banco de Dados: é necessário resolver as questões relativas ao projeto e implementação de um servidor de banco de dados geográfico robusto para suportar a inclusão de todo o conjunto de dados a ser servido, incluindo os dados de observação da Terra e os dados temáticos produzidos a partir deles.
  • Dinâmica no espaço e no tempo: em geral as aplicações geográficas estão preparadas para apresentar e analisar dados em instantes de tempo pré-definidos em uma certa região do espaço. É um desafio nessas aplicações um tratamento da dinâmica dos dados não apenas no espaço, mas também no tempo. A análise de dados de monitoramento ambiental sem considerar a dinâmica no tempo é muito pobre e por isso, no TerraBrasilis visa disponibilizar funcionalidades para explorar a componente temporal dos dados de monitoramento sistemático ao longo do tempo. Por exemplo: além de apenas apresentar o dado do PRODES de um dado ano, pode-se desejar observar uma animação dos desmatamentos detectados ao longo dos últimos 10 anos, em uma determinada região.
  • Interoperabilidade: é necessário que os dados de monitoramento ambiental possam ser acessados via uma arquitetura de serviços, usando os conceitos de Infraestruturas de Dados Espaciais, de maneira que possam ser integrados em outros portais de acesso e organizações como por exemplo, a Infraestrutura Nacional de Dados Espaciais – INDE. A INDE é uma iniciativa do governo brasileiro com o propósito de catalogar, integrar e harmonizar dados geoespaciais existentes nas instituições do governo brasileiro, produtoras e mantenedoras desse tipo de dado, de maneira que possam ser facilmente localizados, explorados e acessados para os mais diversos usos, por qualquer cliente que tenha acesso à Internet.
  • Metadados: é necessário estudar e prover a documentação sintática e semântica dos dados de monitoramento produzidos. A descrição semântica de classes, informações de proveniência, uso pretendido, restrições de uso, entre outras informações, deve ser explícita e parte de mecanismos de busca e seleção. Esses também devem ser oferecidos de acordos com padrões adotados na comunidade de geoinformática.
  • Análises: a organização das bases de dados segundo padrões de interoperabilidades permite a construção de análise mais sofisticadas diretamente no portal de disseminação, usando o TerraBrasilis. A sobreposição de dados de diferentes projetos já é um tipo de análise. O TerraBrasilis deve suportar a definição de áreas de interesse sejam pré-definidas (ex. um certo município) ou criadas por edição pelo usuário. Para uma certa área de interesse podem ser feitas consultas baseadas em operações de álgebra de mapas entre camadas, sumarização de dados em gráficos (barra, torta), e outras que devem ser projetadas de acordo com as particularidades do projeto de monitoramento, .
  • Acesso personalizado por classes de usuários: o TerraBrasilis irá permitir a definição de classes de usuários as quais podem ser usadas por exemplo, para restringir o acesso a dados, ou fornecer análise mais complexas ou mais simplificadas facilitando a usabilidade do sistema.
Finalmente, deve-se notar que todos os problemas e indicações de solução não são úteis somente na disseminação dos dados de monitoramento do Programa Amazônia. Ao contrário, o monitoramento de outros biomas, dentro de outros projetos vai aumentar o volume e a diversidade dos dados processados e também dos dados temáticos produzidos. Por isso, é importante que a questão da gerência de dados e das ferramentas de acesso e disseminação sejam parte de sua concepção, com o devido aporte de profissionais de tecnologia da informação e recursos computacionais.
